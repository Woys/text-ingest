## 2024-05-17 - Optimize JSON Serialization with Pydantic V2
**Learning:** This application heavily relies on Pydantic models to structure and validate its data payload prior to formatting and writing records out to text formats such as JSONL. Using `json.dumps()` over intermediate dictionaries (`model.to_output_dict()`) skips Pydantic's optimized rust-based serialization layer, resulting in significant serialization overhead. However, when migrating away from default Python serialization, special care must be taken with Python specific datatypes such as `date` and `datetime` which have different default string serialization behaviors compared to Pydantic (`iso8601`).
**Action:** When working with Pydantic V2 models, use `model.model_dump_json()` instead of standard `json.dumps()`. To prevent breaking compatibility for `datetime` or `date` formats, use `@field_serializer(..., mode="plain", when_used="json")` to override default behaviour and ensure correct formatting.
